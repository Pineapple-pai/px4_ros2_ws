# px4_ros2_ws 优化路线图

本文档面向当前工作区的真实目标：

- `ROS 2 Humble`
- `Livox Mid360`
- `FAST-LIO`
- `Ego-Planner`
- `PX4`

目标不是回退到旧的 `MAVROS + ROS1` 架构，而是在你当前工作区上，逐步完成：

`Gazebo 仿真闭环 -> 原生 3D 点云 -> FAST-LIO 接入 -> Ego-Planner 重规划 -> 真机 Mid360 统一链路`

---

## 一、最终目标架构

目标链路如下：

`Mid360 / 仿真 Mid360 点云 -> FAST-LIO -> 局部地图 / 里程计 -> Ego-Planner -> PX4 轨迹接口 -> PX4 飞控`

按环境拆分：

### 1. 仿真阶段

- 传感器：Gazebo 中的 `Mid360` 风格 3D 点云
- 定位：
  - 第一阶段可先用 PX4 本地位姿桥接给规划器
  - 第二阶段切换为 `FAST-LIO` 输出
- 规划：`Ego-Planner`
- 控制：`px4_trajectory_interface`

### 2. 真机阶段

- 传感器：`Livox Mid360`
- 定位：`FAST-LIO`
- 规划：`Ego-Planner`
- 控制：`px4_trajectory_interface`

---

## 二、为什么不能原样照旧版执行

旧版思路里有几处不再适合当前工程：

1. 不能把目标改成 `MAVROS + Offboard + ROS1` 风格
- 你当前工作区主线已经是 `ROS 2 + PX4 ROS2 Interface + 自定义桥接`
- 如果改成 `MAVROS` 主链，会引入重复控制链和更高维护成本

2. 不能一开始就同时重构感知、定位、规划、控制四层
- 当前最大风险不是“模块缺失”
- 而是“链路没有稳定闭环”

3. 仿真阶段不应直接以真机参数和真机接线思路推进
- 仿真优先解决的是：
  - 3D 点云是否稳定
  - Ego-Planner 是否稳定出轨迹
  - PX4 是否稳定执行

---

## 三、推荐实施顺序

### 阶段 A：先完成 Gazebo 原生 3D 点云闭环

这是当前最高优先级。

#### 目标

在 Gazebo 中实现：

`原生 3D 点云 -> Ego-Planner -> /planning/position_cmd -> PX4 轨迹跟踪`

#### 当前状态

当前工作区已经有这些基础：

- `ego_planner_offboard.launch.py`
- `px4_trajectory_interface`
- `FAST-LIO` 包
- `Livox ROS2 driver`
- 新增的 `iris_mid360_sim` 仿真模型

#### 本阶段成功判据

需要满足以下条件：

1. Gazebo 输出原生 `PointCloud2`
- 目标话题：`/sim/mid360/points`
- 当前实现优先使用 `px4_gazebo_depth_bridge` 将 Gazebo Classic depth image 转为 `PointCloud2`

2. 规划器收到地图输入
- 目标话题：`/autonomy/local_map`

3. 规划器收到位置输入
- 可先用 `/autonomy/lio_odometry` 的仿真桥

4. 发送目标点后能持续输出轨迹命令
- 目标话题：`/planning/position_cmd`

5. PX4 持续收到轨迹设定点
- 目标话题：`/fmu/in/trajectory_setpoint`

#### 本阶段建议

- 先不要强制把仿真定位切到 `FAST-LIO`
- 先把原生点云和规划控制链闭合
- 重点验证 `/sim/mid360/points` 是否稳定发布

---

### 阶段 B：在仿真中接入 FAST-LIO

当阶段 A 稳定后，再做这一阶段。

#### 目标

在 Gazebo 中实现：

`仿真 Mid360 风格点云 + 仿真 IMU -> FAST-LIO -> Ego-Planner`

#### 推荐做法

1. 使用仿真点云作为 `FAST-LIO` 输入
- 当前建议输入话题改为：
  - `/sim/mid360/points`

2. 使用 PX4 IMU 桥接为 `FAST-LIO` 可用 IMU
- 当前已有：
  - `px4_fastlio_bridge/px4_imu_bridge`

3. 输出通过 `lio_odometry_bridge` 统一给规划器
- 统一输出：
  - `/autonomy/lio_odometry`
  - `/autonomy/local_map`
  - `/autonomy/cloud_registered`

#### 注意事项

- 仿真点云不是 Livox 真正采样模型
- 所以这个阶段的意义是“验证 FAST-LIO 链路能不能在仿真中跑通”
- 不是做最终精度评估

#### 本阶段成功判据

1. `FAST-LIO` 有稳定里程计输出
- `/Odometry`

2. `lio_odometry_bridge` 正常输出
- `/autonomy/lio_odometry`
- `/autonomy/local_map`

3. Ego-Planner 改用 `FAST-LIO` 输出后仍能正常出轨迹

---

### 阶段 C：让 Ego-Planner 成为主避障/绕飞模块

#### 目标

将当前“基于距离触发的刹车/悬停”升级为“基于局部地图的重规划绕飞”。

#### 原则

- 保留当前 `px4_trajectory_interface`
- 不再引入 `MAVROS`
- 让 Ego-Planner 输出的 `position_cmd` 直接进入 PX4 轨迹接口

#### 需要确保的输入输出

Ego-Planner 输入：

- `odom_world`
- `grid_map/odom`
- `grid_map/cloud`
- `/move_base_simple/goal`

Ego-Planner 输出：

- `/planning/bspline`
- `/planning/position_cmd`

PX4 侧输入：

- `/fmu/in/offboard_control_mode`
- `/fmu/in/trajectory_setpoint`

#### 本阶段成功判据

1. 无障碍时直飞目标点
2. 障碍物出现在路径上时自动绕飞
3. 不再只停住或原地悬停

---

### 阶段 D：真机 Mid360 + FAST-LIO + Ego-Planner

当仿真链稳定后，再推进真机。

#### 目标

实现：

`Mid360 -> Livox ROS2 Driver -> FAST-LIO -> Ego-Planner -> PX4`

#### 推荐接入链

1. Mid360 驱动
- 话题：
  - `/livox/lidar`
  - `/livox/imu`

2. FAST-LIO
- 配置文件优先使用当前工作区已有配置：
  - [mid360.yaml](/home/p/px4_ros2_ws/src/px4_fastlio_bridge/config/mid360.yaml:1)
  - [mid360_ego_planner.template.yaml](/home/p/px4_ros2_ws/src/px4_fastlio_bridge/config/mid360_ego_planner.template.yaml:1)

3. LIO 输出桥接
- 使用 `lio_odometry_bridge`

4. Ego-Planner 订阅桥接后的 odom/map

5. PX4 继续使用当前 `trajectory_interface`

#### 真机阶段的重点

- 外参标定
- 时间同步
- Mid360 安装位姿
- FAST-LIO 参数整定
- Ego-Planner 安全速度和安全距离

## 七、当前结论

### 2026-06-29 仿真闭环进展

当前工作区已经跑通仿真链路：

`Gazebo iris_mid360_sim -> /sim/mid360/points + /fmu/out/sensor_combined -> FAST-LIO -> /autonomy/lio_odometry + /autonomy/local_map -> Ego-Planner -> /planning/position_cmd -> /fmu/in/trajectory_setpoint`

关键修正：

- `px4_imu_bridge` 已支持 PX4 `SensorCombined`，仿真默认从 `/fmu/out/sensor_combined` 生成 `/sim/imu`
- `ego_planner_offboard.launch.py` 的 `use_depth_camera_fastlio:=true` 路径已切到 `/sim/mid360/points` + `/sim/imu`
- `scripts/start_autonomy_with_planning.sh` 默认使用 `iris_mid360_sim`、FAST-LIO 和 Ego-Planner，不再默认用 PX4 local position 仿真桥覆盖 `/autonomy/lio_odometry`

已验证输出：

- `/sim/mid360/points`：Gazebo Mid360 风格点云
- `/sim/imu`：PX4 IMU 桥接输出
- `/Odometry`：FAST-LIO 里程计
- `/autonomy/lio_odometry`：规划侧里程计
- `/autonomy/local_map`：规划侧局部点云地图
- `/planning/position_cmd`：Ego-Planner 轨迹命令
- `/fmu/in/trajectory_setpoint`：PX4 轨迹设定点

推荐一键启动：

```bash
./scripts/start_autonomy_with_planning.sh
```

推荐一键验证：

