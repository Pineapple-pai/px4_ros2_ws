# “Go to location + Navigation2 (Nav2)” 完整实现方案

本方案基于 **ROS 2 Navigation Stack (Nav2)**，为 `px4_ros2_ws` 项目提供完整的自主避障导航能力。Nav2 是 ROS 2 官方导航框架，内置全局规划器（A*、NavFn）和局部规划器（DWB、Regulated Pure Pursuit），可开箱即用。

### 3.2 点云转栅格：将 3D 点云转为 2D 地图

Nav2 需要 `nav_msgs/OccupancyGrid` 类型的 `/map` 话题作为全局地图。FAST-LIO2 发布的是 3D 点云（`/fastlio2/world_cloud` 或 `/cloud_map`），需要转换为 2D 占据栅格。

#### 方案 A：使用 octomap_server（推荐）

创建 `fastlio2_nav2/launch/octomap_2d_launch.py`：

```python
from launch import LaunchDescription
from launch_ros.actions import Node

def generate_launch_description():
    return LaunchDescription([
        Node(
            package='octomap_server',
            executable='octomap_server_node',
            name='octomap_server',
            output='screen',
            parameters=[{
                'frame_id': 'lidar',  # 与 FAST-LIO2 的 frame_id 一致
                'pointcloud_min_z': -0.3,   # 忽略地面（雷达安装高度 0.5m）
                'pointcloud_max_z': 1.5,    # 忽略天花板
                'occupancy_min_z': -0.3,
                'occupancy_max_z': 1.5,
                'resolution': 0.05,
            }],
            remappings=[
                ('cloud_in', '/fastlio2/world_cloud'),  # 根据实际话题名调整
                ('projected_map', '/map'),              # 输出 2D 栅格地图
            ]
        )
    ])
```

> **注意**：先运行 `ros2 topic list -t` 确认 FAST-LIO2 的点云话题名，可能是 `/fastlio2/world_cloud` 或 `/cloud_map`。

#### 方案 B：使用 pointcloud_to_laserscan（备选）

如果只想用 2D 激光数据：

```bash
sudo apt install ros-humble-pointcloud-to-laserscan
```

创建转换节点，将点云转为激光扫描，再交由 Nav2 的 `costmap_2d` 处理。

### 3.3 LIO 里程计桥接：FAST-LIO2 → Nav2 标准里程计

FAST-LIO2 输出的位姿是 `camera_init → body`（传感器视角），而 Nav2 需要的是 `odom → base_footprint`（机器人底盘视角）。需要桥接节点转换。

#### 创建 `lio_odom_bridge` 节点

在 `px4_autonomy_mode/src/` 下创建 `lio_odom_bridge.py`：

```python
#!/usr/bin/env python3
import rclpy
from rclpy.node import Node
from nav_msgs.msg import Odometry
from geometry_msgs.msg import TransformStamped
import tf2_ros

class LIOOdomBridge(Node):
    def __init__(self):
        super().__init__('lio_odom_bridge')
        # 订阅 FAST-LIO2 的里程计
        self.sub = self.create_subscription(
            Odometry, 
            '/Odometry',  # FAST-LIO2 的里程计话题
            self.odom_callback, 
            10
        )
        # 发布标准里程计（供 Nav2 使用）
        self.pub = self.create_publisher(
            Odometry, 
            '/odom',  # Nav2 默认订阅的里程计话题
            10
        )
        # TF 广播
        self.tf_broadcaster = tf2_ros.TransformBroadcaster(self)

    def odom_callback(self, msg):
        # 直接转发（实际项目中可能需要坐标系转换）
        self.pub.publish(msg)
        
        # 广播 TF: odom → base_link
        t = TransformStamped()
        t.header.stamp = msg.header.stamp
        t.header.frame_id = 'odom'
        t.child_frame_id = 'base_link'
        t.transform.translation.x = msg.pose.pose.position.x
        t.transform.translation.y = msg.pose.pose.position.y
        t.transform.translation.z = msg.pose.pose.position.z
        t.transform.rotation = msg.pose.pose.orientation
        self.tf_broadcaster.sendTransform(t)

def main(args=None):
    rclpy.init(args=args)
    node = LIOOdomBridge()
    rclpy.spin(node)

if __name__ == '__main__':
    main()
```

> **关键**：Nav2 需要的 TF 树结构为 `map → odom → base_footprint → base_link → livox_frame`。FAST-LIO2 的里程计必须正确映射到 `odom → base_footprint`。

### 3.5 Nav2 → PX4 桥接节点

Nav2 默认输出速度指令（`/cmd_vel`），但 PX4 Offboard 模式需要**位置设定点**。需要桥接节点将速度积分转换为位置。

#### 创建 `nav2_to_px4_bridge.py`

在 `px4_autonomy_mode/src/` 下创建：

```python
#!/usr/bin/env python3
import rclpy
from rclpy.node import Node
from geometry_msgs.msg import Twist, PointStamped, PoseStamped
from nav_msgs.msg import Odometry
import math

class Nav2ToPX4Bridge(Node):
    def __init__(self):
        super().__init__('nav2_to_px4_bridge')
        
        # 订阅 Nav2 的速度指令
        self.sub_cmd = self.create_subscription(
            Twist, 
            '/cmd_vel', 
            self.cmd_vel_callback, 
            10
        )
        
        # 订阅当前位姿（FAST-LIO2）
        self.sub_odom = self.create_subscription(
            Odometry,
            '/Odometry',
            self.odom_callback,
            10
        )
        
        # 发布位置设定点给 px4_autonomy_mode
        self.pub_target = self.create_publisher(
            PointStamped,
            '/autonomy/target_ned',
            10
        )
        
        self.current_pose = None
        self.integrated_x = 0.0
        self.integrated_y = 0.0
        self.last_time = None

    def odom_callback(self, msg):
        self.current_pose = msg.pose.pose
        # 重置积分位置为当前位置
        if self.last_time is None:
            self.integrated_x = self.current_pose.position.x
            self.integrated_y = self.current_pose.position.y

    def cmd_vel_callback(self, msg: Twist):
        if self.current_pose is None:
            return
        
        # 速度积分得到位置偏移（简化的积分方式）
        now = self.get_clock().now()
        dt = 0.1  # 固定时间步长，实际应从时间戳计算
        
        self.integrated_x += msg.linear.x * dt
        self.integrated_y += msg.linear.y * dt
        
        # 发布目标点
        target = PointStamped()
        target.header.frame_id = 'ned'
        target.header.stamp = now.to_msg()
        target.point.x = self.integrated_x
        target.point.y = self.integrated_y
        target.point.z = self.current_pose.position.z  # 保持高度
        
        self.pub_target.publish(target)

def main(args=None):
    rclpy.init(args=args)
    node = Nav2ToPX4Bridge()
    rclpy.spin(node)

if __name__ == '__main__':
    main()
```

> **更稳健的方案**：将 Nav2 输出的路径（`/plan`）直接转换为一系列航点，逐个发送给 `px4_autonomy_mode`，避免积分漂移。

   ```bash
   cd ~/px4_ros2_ws
   bash scripts/start_nav2_autonomy.sh
   ```


3. **确认地图**：
   ```bash
   ros2 topic echo /map --once
   ```
   应有 OccupancyGrid 数据输出。

### 4.2 执行任务

1. 使用遥控器在 PX4 Position 模式下起飞并悬停
2. 在 QGC 中切换到 Offboard 模式
3. 在 QGC 地图上点击 **"Go to location"**

### 4.3 避障与重规划流程

1. **FAST-LIO2** 实时提供位姿与 3D 点云
2. **octomap_server** 将 3D 点云投影为 2D 占据栅格
3. **Nav2 全局规划器**（A*）在栅格地图上规划全局路径
4. **Nav2 局部规划器**（DWB）实时检测障碍物并动态重规划
5. **桥接节点**将速度指令转换为位置设定点
6. **px4_autonomy_mode** 将设定点发送给 PX4 执行

## 七、常见问题与解决方案

### Q1: FAST-LIO2 里程计无法接入 Nav2
- **原因**：坐标系不匹配
- **解决**：使用 `lio_odom_bridge.py` 桥接节点，确保发布 `odom → base_link` 的 TF

### Q2: 没有 /map 话题
- **原因**：点云未正确转换为栅格地图
- **解决**：确认 `octomap_server` 正常运行，检查 `cloud_in` 重映射是否正确

### Q3: Nav2 规划失败
- **原因**：目标点在地图外，或地图未正确加载
- **解决**：检查 `global_frame` 是否为 `map`，确认目标点在栅格地图范围内

### Q4: PX4 无法进入 Offboard 模式
- **原因**：未持续发送设定点
- **解决**：确保桥接节点持续发布 `/autonomy/target_ned`，频率不低于 10Hz

### Q5: 避障响应慢
- **原因**：局部规划器参数不合适或算力不足
- **解决**：降低 `max_vel_x`，减小 `sim_time`，或使用性能更好的机载计算机

## 八、总结

| 对比项 | 自定义方案 (A* + EGO-Planner) | **Nav2 方案** |
|--------|------------------------------|---------------|
| **开发工作量** | 大（需自行实现 A*、点云转栅格） | **小（开箱即用）** |
| **稳定性** | 依赖实现质量 | **经过充分测试** |
| **功能丰富度** | 基础 | **完整（重规划、恢复行为、定位）** |
| **社区支持** | 有限 | **丰富** |
| **适用场景** | 对算力敏感 | **通用** |

**核心优势**：
1. **开箱即用**：Nav2 提供完整的全局规划 + 局部规划 + 行为树管理
2. **成熟稳定**：ROS 2 官方维护，经过大量验证
3. **社区支持**：丰富的文档和案例
4. **可扩展**：支持插件化定制

**关键要点**：
1. **点云转栅格**是 Nav2 能工作的前提
2. **里程计桥接**解决 FAST-LIO2 与 Nav2 的坐标系语义差异
3. **速度→位置转换**是 Nav2 输出与 PX4 Offboard 模式匹配的关键
4. **安全第一**：始终保留遥控器手动接管能力
